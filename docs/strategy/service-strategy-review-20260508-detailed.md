## 1. 현재 접근법의 전제와 사각지대

### 현재 전제 (Phase 1~5 로드맵)
```
Phase 1: STT/LLM/TTS ✅
Phase 2: OCR/Vision ← 현재
Phase 3: 웹검색 Agent
Phase 4: 인증/구독
Phase 5: 배포 패키지
```

### 핵심 전제 문제

**전제 1: "기기에 접근하면 쓸 수 있다"**
→ 사각지대: 시각장애인이 이 서비스가 존재하는지 어떻게 알까?
  - 앱스토어 검색 → 화면 못 봄
  - 키오스크 → 어디에 있는지 어떻게 찾나
  - 웹 → URL을 어떻게 입력하나

**전제 2: "스페이스바 한 개면 충분하다"**
→ 사각지대: 스마트폰에는 스페이스바가 없음
  - 모바일 앱 → 버튼 위치를 어떻게 찾나
  - 키오스크 → 터치 위치를 어떻게 찾나
  - 해결책: 화면 어디를 터치해도 작동하는 "전체화면 터치"

**전제 3: "말로 묻고 말로 답하면 된다"**
→ 사각지대: 소음 환경 (버스, 지하철, 병원 대기실)
  - STT 인식률 급감
  - 개인정보 발화 꺼려짐
  - 해결책: 이어폰 감지 + 진동 피드백 모드

**전제 4: "OCR/Vision이 핵심이다"**
→ 사각지대: 실제 시각장애인의 가장 빈번한 니즈는?
  - 문서 읽기보다 길 찾기, 대중교통, 약병 확인이 더 급함
  - 카메라로 실시간 촬영 → 즉시 읽기가 더 중요할 수 있음

## 2. 2025-2026 업계 변화가 흔드는 것들

### 변화 1: GPT-4o / Gemini 실시간 음성 대화
- OpenAI의 실시간 음성 대화 → 별도 STT/TTS 파이프라인 불필요
- **우리에게 의미**: 로컬 Ollama 파이프라인의 속도 불이익이 커짐
- **대응**: Claude API fallback 강화, 온프레미스 버전 차별화

### 변화 2: 스마트폰 내장 접근성 기능 강화
- iOS VoiceOver, Android TalkBack 점점 강력해짐
- **우리에게 의미**: "있으면 쓰겠지" 수준의 경쟁자가 생김
- **대응**: 특화 기능에 집중 (OCR+분석, 공공기관 문서, 한국어 최적화)

### 변화 3: 공공기관 디지털 접근성 의무화 강화
- 2024년 장애인차별금지법 개정 → 키오스크 접근성 의무
- **기회**: B2G (정부 납품) 시장이 열림
- **대응**: Phase 5 배포에서 공공기관 버전 우선

### 변화 4: AI PC / 로컬 AI 시대
- MS Copilot+ PC, Apple Intelligence → 온디바이스 AI 보편화
- **기회**: 우리의 로컬 Ollama 방식이 오히려 트렌드에 부합
- **대응**: "인터넷 없이도 작동" 강점으로 마케팅

## 3. 시각장애인이 처음 접근하는 순간 시나리오

### 시나리오 A: 스마트폰 앱
```
문제: 앱스토어에서 어떻게 찾나?
→ 앱 이름을 모름 → 검색 불가

해결책:
1. 스크린리더(VoiceOver/TalkBack) 호환 앱스토어 등록
2. 앱 설명에 "시작하려면 어디든 탭하세요" 명시
3. 앱 아이콘에 점자 라벨 가이드 제공
4. 카카오톡/문자로 앱 링크 공유 받아서 설치
5. 사회복지사/보조인이 설치해주는 경로
```

### 시나리오 B: 키오스크/민원발급기
```
문제: 기기 앞에 섰을 때 이 기기가 뭔지 모름

해결책:
1. 기기 접근 시 자동 음성 안내 (모션 감지)
   "안녕하세요. AI 음성 도우미입니다. 어디든 터치하시면 시작해요."
2. 점자 안내 스티커 부착 위치 표준화
3. NFC 태그 → 스마트폰으로 서비스 연결
4. QR코드 → 스마트폰으로 연결 (하지만 QR을 못 봄 → NFC 우선)
```

### 시나리오 C: 웹 서비스 (SaaS)
```
문제: URL을 어떻게 입력하나

해결책:
1. 사회복지관/맹학교 통한 배포
2. 국가보조기기 등록 → 공식 채널
3. 기억하기 쉬운 주소 (예: ai.voice.kr)
4. 전화번호로 연결 가능한 서비스
```

## 4. 개발 방향 수정 제안

### 즉시 수정 (Phase 2 완료 전)
1. **전체화면 터치 = 스페이스바 동작** 추가
   - 모바일 대응 필수
2. **앱 첫 실행 시 자동 안내** 강화
   - "안녕하세요. AI 음성 도우미예요. 어디든 터치하시거나 스페이스바를 누르시면 시작해요."
3. **카메라 실시간 촬영 모드** 추가
   - 파일 업로드보다 카메라가 실제 사용 빈도 높음

### Phase 3 수정 (웹검색 → 실용성 강화)
```
기존: 웹검색 Agent
수정: 실생활 특화 기능 우선
  - 약병/처방전 촬영 즉시 읽기
  - 버스/지하철 노선도 읽기
  - 메뉴판 읽기
  - 영수증 읽기
  - 공공기관 문서 읽기
```

### Phase 4 수정 (인증/구독 → B2G 우선)
```
기존: 무료→유료 전환
수정: 
  - B2G (공공기관): 키오스크, 민원발급기
  - B2B (복지관, 맹학교): 기관 라이선스
  - B2C (개인): 추후 (시장 검증 후)
```

### Phase 5 수정 (배포 패키지 구체화)
```
우선순위 순서:
1. 공공기관 키오스크 버전 (가장 큰 시장)
2. 복지관/병원 미니PC 버전
3. 개인 스마트폰 앱
4. SaaS (웹)
5. 가정용 EXE
```

## 5. 서버 사업성 분석 (온프레미스 vs SaaS)

### 온프레미스 (A5000×4, L40S×4)
```
A5000 ×4: ~200명 동시 접속, 월 $800
L40S ×4: ~500명 동시 접속, 월 $2,000

적합 대상:
- 공공기관 (데이터 외부 반출 불가)
- 병원 (의료정보 보안)
- 대기업 복지 시스템
```

### SaaS (Claude API + 로컬 Fallback)
```
Claude Haiku: 1,000토큰 = $0.00025
월 100만 요청 = 약 $250
적합 대상: 일반 사용자, 소규모 기관
```

## 6. 경쟁사/유사 서비스 현황

| 서비스 | 특징 | 약점 |
|--------|------|------|
| iOS VoiceOver | 내장, 무료 | 범용, 한국어 약함 |
| JAWS (미국) | 전문 스크린리더 | 고가, 영어 중심 |
| 센스리더 (국내) | 한국어 스크린리더 | OCR 약함, 고가 |
| 설리번+ (국내) | 시각장애인 앱 | 기능 제한적 |
| **우리** | 로컬 AI+OCR | **개발 중** |

**우리의 차별점:**
- 로컬 처리 (개인정보 보호)
- 고품질 한국어 OCR
- 공공문서 특화
- 말로만 모든 것 가능
