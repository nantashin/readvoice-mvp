# READ VOICE Pro — 서비스 전략 재검토
> 작성: 2026-05-08 | 검토자: Claude Sonnet 4.5

## 1. 현재 접근법의 전제와 사각지대

### 핵심 전제 문제

**전제 1: 기기에 접근하면 쓸 수 있다**
- 사각지대: 시각장애인이 서비스 존재를 어떻게 알까?
- 앱스토어 검색, 키오스크 위치, URL 입력 모두 문제

**전제 2: 스페이스바 한 개면 충분하다**
- 사각지대: 스마트폰에는 스페이스바가 없음
- 해결책: 전체화면 터치 = 스페이스바 동작

**전제 3: 말로 묻고 말로 답하면 된다**
- 사각지대: 소음 환경 (버스, 지하철, 병원)
- 해결책: 이어폰 감지 + 진동 피드백 모드

**전제 4: OCR/Vision이 핵심이다**
- 사각지대: 실제 니즈는 약병, 버스노선, 메뉴판이 더 급함
- 해결책: 카메라 실시간 촬영 모드 추가

## 2. 2025-2026 업계 변화

**변화 1: GPT-4o/Gemini 실시간 음성 대화**
- 대응: Claude API fallback 강화, 온프레미스 차별화

**변화 2: 스마트폰 내장 접근성 기능 강화**
- 대응: 특화 기능 집중 (OCR, 공공기관 문서, 한국어)

**변화 3: 공공기관 디지털 접근성 의무화**
- 기회: B2G 시장
- 대응: Phase 5에서 공공기관 버전 우선

**변화 4: AI PC / 로컬 AI 시대**
- 기회: 로컬 Ollama 방식이 트렌드에 부합
- 대응: 인터넷 없이도 작동 강점 마케팅

## 3. 시각장애인 첫 접근 시나리오

**스마트폰 앱:**
- 스크린리더 호환 앱스토어 등록
- 카카오톡/문자로 링크 공유
- 사회복지사 설치 경로

**키오스크/민원발급기:**
- 모션 감지 자동 음성 안내
- NFC 태그로 스마트폰 연결
- 점자 안내 스티커 표준화

**웹 서비스:**
- 사회복지관/맹학교 통한 배포
- 기억하기 쉬운 주소 (예: ai.voice.kr)

## 4. 개발 방향 수정 제안

**Phase 2 완료 전 즉시 추가:**
- 전체화면 터치 = 스페이스바 동작
- 카메라 실시간 촬영 버튼
- 앱 첫 실행 자동 안내 강화

**Phase 3 수정 (웹검색 → 실생활 특화):**
- 약병/처방전 촬영 즉시 읽기
- 버스/지하철 노선도 읽기
- 메뉴판, 영수증, 공공문서 읽기

**Phase 4 수정 (B2G 우선):**
- 1순위: 공공기관 키오스크
- 2순위: 복지관/병원 미니PC
- 3순위: 개인 스마트폰 앱

**Phase 5 배포 우선순위:**
1. 공공기관 키오스크 버전
2. 복지관/병원 미니PC 버전
3. 개인 스마트폰 앱
4. SaaS (웹)
5. 가정용 EXE
